superset 源码/pip安装+superset二次开发环境搭建。 集成echart

环境： mac 10.15 Catalina + Anaconda 3 + Python3.7 + superset 0.30

17装过一次，今年又开始使用supeset了。遇到些问题，所以这次记录下

两种方式都是在Anaconda虚拟python环境中进行。 源码安装superset,和pip安装superset

一.源码安装superset 1.首先从github 下载源码包 git clone https://github.com/apache/incubator-superset.git cd incubator-superset 2.安装扩展依赖，及开发依赖 pip install -r requirements.txt -i https://pypi.douban.com/simple/ 如果只是安装superset不用安装dev的依赖，如果是二次开发需要安装 pip install -r requirements-dev.txt -i https://pypi.douban.com/simple/

（源代码里superset\static\assets这个软连接可以在linux或者Mac上正常工作，但是在windows下不能正常工作，需要删除这个文件夹，然后重新链接 mklink /J “C:XXXX\incubator-superset\superset\static\assets” “C:XXXX\incubator-superset\superset\assets” )

4.安装开发者模式 .安装superset

在 incubator-superset 目录执行

pip install -e .

5.开始配置superset

创建管理员账户和密码 这个命令好像已经不推荐使用了。 fabmanager create-admin –app superset 推荐使用 flask fab create-admin

*以下操作要进入superset/bin目录下

到此为止superset启动成功。可以通过浏览器访问

下面开始二次开发的配置。

安装nodejs,官网下载安装就OK，然后node -v 查看版本 https://nodejs.org/en/

安装cnpm淘宝镜像 npm install -g cnpm –registry=https://registry.npm.taobao.org

12、编译前端文件，并进入前端开发者模式

以下操作要进入superset/assets目录下

安装编译所需模块 使用淘宝镜像安装 cnpm install -d 或 npm install -d

window会遇到的问题：修改package.json的第18行（这里是设置变量，但是此命令在linux和Mac是可以通过的，windows不能通过），在NODE_ENV前面添加set，即"build”: “set NODE_ENV=production webpack –mode=production –colors –progress”,

编译前端文件，并进入前端开发者模式 npm run build npm run dev

运行完npm run dev，命令窗口会停止，不要误以为是执行不下去，保持窗口开启状态即可，一有文件变动时，它会重新编译 13、把项目导入到pyCharm File -> New Project -> Location选择incubator-superset所在目录 -> 选择Existing interpreter -> 填写incubator-superset\venv\Scripts\python.exe所在位置

14、启动项目 在 incubator-superset/superset 目录下新建 run.py，添加如下内容： from superset import app app.run(debug=True, host='0.0.0.0’, port=8088)

等 pycharm 右下角 processes running 完成后就可以debug run.py了。访问：http://localhost:8088,输入管理员账号跟密码，登录。
